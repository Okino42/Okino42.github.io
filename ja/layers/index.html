アーキテクチャ段階

最小の動くループから始めて、安全性、永続状態、バックグラウンド実行、マルチエージェント協調、外部 capability 連携へと進む。

読み方

最初に読むなら

第1段階から始めます。後のすべての能力はコアループの外側に積み上がるため、そこを飛ばさない方がよいです。

読み方

途中でぼやけたら

まず今の章がどの段階に属するかを確認し、それが安全性・状態・ランタイム・基盤のどれを足しているのかを見ます。

読み方

自作したいなら

各段階の終わりごとに、1つの実装可能なシステムの塊が手元に残るべきで、単なる用語集になってはいけません。

基礎ループ補助資料

`s01-s06` を読む前に、主ループ・tool 入口・データ構造境界を先に安定させる

最初の6章で大事なのは個別機能ではなく、最小ループがどう成立するかです。ユーザー入力がどう入り、ツール結果がどう戻り、どんな状態容器があるかを先に押さえます。

アーキテクチャ全体図

全体像を先に見てから現在の章へ戻るための俯瞰図です。

強化段階で混ざりやすいのは個別機能の難しさではなく、権限・hook・memory・prompt・recovery がすべて制御面で交わる点です。

P1 コアループ

コアループ: ツール、計画、委譲、スキル、コンテキスト圧縮を含む最小の単一エージェントを作る段階。

この段階を終えたら、動く単一 agent harness を自力で書けるはずです。

段階の収束ポイント

ここで一度止まり、単一 agent の背骨を自分で作り直す

この層の最後の章を読んだら、そのまま先へ走るのではありません。入口へ戻り、この層を自分で作り直してから次へ進む方が安定します。

段階の入口s01段階の収束章s06

この層で今なら自分で作り直せるべきもの

複数ターン継続でき、tool を呼び、todo plan を持ち、単発 subtask を委譲し、skill を必要時だけ読み込み、最小限の compact を行える単一 agent harness。
