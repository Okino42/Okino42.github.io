学習パス

s01 から s19 まで、4段階で agent システムを積み上げる

このページの使い方

初回の通読

まずは上から順に読む。最初の6章が主ループで、後半はその上に積まれています。

途中で混ざり始めたら

先にコードへ潜らず、この章がどの段階に属するかを見て、bridge doc で task・runtime・teammate・worktree の境界を補正します。

自分で実装するなら

各段階が終わるたびに最小版を自分で書き直す。一気に s19 まで進んでからまとめて補わない。

何度も戻る価値のある橋渡し資料

中盤以降で混線し始めたら、次の章へ突っ込む前にまずここへ戻ります。

クエリ制御プレーンツール実行ランタイムランタイムタスクモデルチームメイト・タスク・レーンモデルMCP 能力層マップ

このタイムラインは順序だけでなく、どこで止まるべきかも示す

各段階のあとで 1 回作り直してから次の段階へ入る

ただ下へ読み進めるだけだと、章境界はいつか必ずぼやけます。`s06 / s11 / s14 / s19` で止まり、その段階で成立した system slice を作り直せるか確認する方が安定します。

s01-s06

ここで一度止まり、単一 agent の背骨を自分で作り直す

この時点で作り直せるべきもの

複数ターン継続でき、tool を呼び、todo plan を持ち、単発 subtask を委譲し、skill を必要時だけ読み込み、最小限の compact を行える単一 agent harness。

段階の収束点を開く: s06
s07-s11

ここで制御面を安定させてからタスク実行層へ入る

この時点で作り直せるべきもの

ただ動くだけでなく、実行前 gate、拡張 hook、長期 memory、入力組み立て、recovery 分岐までそろった安定した single agent。

段階の収束点を開く: s11
s12-s14

ここで work runtime を作り切ってから team 層へ進む

この時点で作り直せるべきもの

永続 task goal を持ち、遅い仕事を background で回し、notification で結果を戻し、時間で新しい仕事を起動できる最小 runtime system。

段階の収束点を開く: s14
s15-s19

最後の段階で作るのは機能の山ではなくプラットフォーム境界です

この時点で作り直せるべきもの

永続 teammate、共有 protocol、自律 claim、分離 execution lane、そして native/external capability を 1 つの control plane へ戻したプラットフォームの骨格。

段階の収束点を開く: s19
