基礎ドキュメント

アーキテクチャ全体図

全体像を先に見てから現在の章へ戻るための俯瞰図です。

なぜこの章順なのか

なぜこの順序で学ぶのか、順番を崩すと何が混乱するのかを整理する資料です。

コード読解順

ローカルの `agents/*.py` をどの順で開き、各章でまずどの状態や関数を見るべきかを整理した読解ガイドです。

参照モジュール対応表

参照リポジトリの高信号モジュール群と現在の教材章を対応付け、章順の妥当性を確認する地図です。

教材の守備範囲

この教材が意図的に省いている範囲を示し、主線を守るための資料です。

用語集

用語が増えて混ざり始めたときに戻る境界整理用の用語集です。

主要データ構造マップ

主要な record 構造を横断的に見直すための資料です。

エンティティ地図

オブジェクトとモジュール関係から全体を再確認する地図です。

深掘りドキュメント

クエリ制御プレーン

1つの要求が control plane をどう通るかを通しで補う資料です。

1 リクエストのライフサイクル

1回の要求を入口から write-back まで通して確認する補助資料です。

クエリ遷移モデル

エラー回復・文脈圧縮・予算制御・hook が重なり始めたときに、query がなぜ次のターンへ続くのかを補う資料です。

ツール制御プレーン

ツール呼び出しが共通の実行面に入る流れを補う資料です。

ツール実行ランタイム

tool の並列実行と直列実行、progress 更新、結果順序、context 統合をまとめて補う資料です。

メッセージと Prompt の組み立てパイプライン

message と prompt 片をどの順に組み立てるかを補う解説です。

ランタイムタスクモデル

作業目標・実行記録・通知の3層境界をまとめて補う資料です。

チームメイト・タスク・レーンモデル

teammate・protocol request・task・runtime slot・worktree lane の 5 層境界を整理します。

MCP 能力層マップ

native tool・plugin・MCP server が 1 つの capability bus へ戻る全体像を補います。
